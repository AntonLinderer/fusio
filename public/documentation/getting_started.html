Welcome, this is a quick introduction to help you get started using this API. On the left sidebar you see all available endpoints. Each endpoint shows the available request methods and the request/response data format.

The API uses OAuth2 for authorization. In order to obtain an access token you have to send an client credentials request to the following endpoint. The app key and secret must be obtained by the API provider.

POST /authorization/token HTTP/1.1 User-Agent: Awesome-App 0.1 Content-Type: application/x-www-form-urlencoded Authorization: Basic YWRtaW46dGVzdDEyMw== Content-Length: 49 grant_type=client_credentials&scope=authorization
HTTP/1.1 200 OK X-Powered-By: psx Content-Length: 148 Content-Type: application/json { "access_token": "0536a0b1fc599246743649991b133eb6dd8889cba59f84d1ff1546ff9b3d8781", "token_type": "bearer", "expires_in": 1433961485 }

The endpoint returns an access token if the authentication was successful. This token can only be used to access the authorization endpoints which give you the option to revoke the access token or get informations about who you are. To request other endpoints you need to specify another scope. The available scopes can be obtained by the API provider. In the following an example to check whether the authorization was successful.

GET /authorization/whoami HTTP/1.1 User-Agent: Awesome-App 0.1 Connection: Keep-Alive Authorization: Bearer 1ab55cbf3eaedd86d87547111dfe2e8d364ff9e0871b930663e28564ef031f65
HTTP/1.1 200 OK X-Powered-By: psx Content-Length: 65 Content-Type: application/json { "id": "1", "status": "1", "name": "Administrator" }
